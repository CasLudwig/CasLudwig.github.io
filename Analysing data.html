Data analysis involves several steps:

  1. Convert Eyelink Data File (*.edf) to an ASCII file:
    • Open a new terminal window (right-click in the terminal that you used to open Matlab and select 'New Tab').
    • Go to the directory in which the to-be-converted files are stored (e.g. ‘cd mydata/experiment1’ [ENTER])
    • Type: ‘edf2asc myedffile.edf –ns’ (without quotes) and hit ENTER. Note 1: use of wildcards before ‘.edf’ is permitted, so to convert all edf files in one folder, simply type ‘edf2asc *.edf…’. Note 2: The ‘-ns’ argument ensures that only events are written to the ASCII file, as opposed to each recorded sample (which results in extremely large files).
  1. Open Matlab and go to the folder in which your ASCII files are stored. The first step is to gain access to the ASCII files you want to process.
    • Type: ‘f=dir(‘myfiles*.asc’)’. I have illustrated the use of a wildcard here, because typically you will want to convert the files from a given participant and all his/her file names will start similarly (e.g. P1_1, P1_2, etc. so to convert all files from P1 you call dir(‘P1*.asc’). The command returns an array of structures. Thus, each element of the array contains 1 structure, corresponding to one ASCII file. The structure contains information such as the filename, date it was created, size, etc. Note the array (called ‘f’ in my example) should have as many cells as there are ASCII files you want to convert. If it has not, you should check what is going wrong.
    • There is a Matlab program called ‘ExtractAllSaccades’ that does what it says on the tin: It will cycle through a series of ASCII files and extract all the saccades (all the information after an ‘ESACC’ message in the ASCII file). The program is called by typing ‘out=ExtractAllSaccades(f,0/1)’, where f is the structure array and the second parameter indicates whether to include trials on which no saccade was made in between display onset and the time out (set to 1 if you wish to skip such trials, as will be the case in most circumstances). Click on ExtractAllSaccades.m to view/download the code of one particular incarnation of this matlab program. Note that this code will only work if the original experiment was programmed in a way that is consistent with the types of "messages" the matlab program expects to see.
    • The program outputs a matrix (which I have named ‘out’ in the example above) in which each row corresponds to one saccade. The number of columns will depend partly on the complexity of the trial identifier. An example trial identifier contains: the trial number, a target angle, SOA. In the ASCII file this will look like: TRIALID 2 45 10. If this is what your trial identifier looks like, each of these three numbers will take up its own column in the output. The ordering of the columns will typically be something like:
    • BLOCK NUMBER – TRIAL IDENTIFIER(S) – LATENCY (with respect to display onset) – DURATION – X START – Y START – X END – Y END – AMPLITUDE – PEAK VELOCITY.
    • There are various incarnations of the ‘ExtractAllSaccades’ program, suited for the peculiarities of individual experiments (e.g. when a manual response is required we might need to extract the reaction time of the button press). Once you have a version of the program that works for your studies, it is likely you will adapt it yourself as you go along and you get to learn more and more about what it is doing (so you learn its short comings).
  1. Now you are alone… That is, from this point onwards you have the data in an easy to manipulate format (i.e. a matrix). You can copy this matrix to Excel or any other spreadsheet program and perform the subsequent processing in that environment. Alternatively, the recommended option – especially if you are planning to run several experiments – is to write some additional Matlab code that goes through the output matrix and selects the saccades that are of critical interest. Whichever way you go about it, you are likely to want to discard certain trials. For instance, if the observer was not fixating properly or moved his/her eyes too early (in anticipation of some critical event), etc. It is straightforward to test against such criteria (e.g. improper fixation can be gauged by calculating the distance between the saccade start point, using the X START and Y START columns, and the known location of the fixation point).
  2. Once you have filtered your data in this way and are satisfied that you have got the set of responses that are of critical interest, you can do the principal analysis that aims to answer whatever question you asked in the first place (e.g. comparing mean saccade latencies in different conditions)!
