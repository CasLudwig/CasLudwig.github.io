These notes are largely for my (our) own reference, but they may be useful to other people.

 

System specifications

 

  • Intel Core 2Quad CPU Q9500 @ 2.83 GHz x 4.  

  • 4 Gb RAM. 

  • NVIDIA GeForce GT430 1 Gb graphics card. 

  • Linux Mint 18 Sarah (based on Ubuntu 16.04)

  • Matlab R2017b

 

 

OS

 

  • Install Linux Mint from cd. I'm using Linux Mint 18 "Sarah" - this seems to be based on Ubuntu 16.04.

  • Edit connections

    • Wired connection 1 (to the world): check whiteboard in lab.

    • Wired connection 2 (to operator PC): check whiteboard in lab.

  • Check the driver manager and adopt the proprietary driver for your graphics card.

 

Matlab

 

  • Download Matlab from Mathworks website.

  • Extract the downloaded files.

  • Open terminal, navigate to the folder that contains the extracted files and run sudo ./install

  • The installer will ask for your Mathworks account log-in details.

  • You will need to activate Matlab, but the installer will guide you through this.

 

PTB

 

  • Follow instructions from the PTB website (choose neurodebian installation).

  • Had to add the ptb path manually (having started matlab as sudo). I remember not having to do this in the past. Who knows...

  • Run PsychLinuxConfiguration.

 

ViewPixx

 

  • Grab 'Software Tools for MAC' folder from the ViewPixx software cd and put it somewhere temporarily (e.g. desktop).

  • Open the folder.

  • Using sudo cp -avr, copy the VPixx_Software_Tools to /usr/share/.

  • Copy (as sudo) 'Datapixx.mexa64' from the 'VPixx_Software_Tools/DatapixxToolbox_trunk/DatapixxToolbox/mexdev/build/matlab/linux64' folder to 'psychtoolbox-3/PsychHardware/DatapixxToolbox/DatapixxBasic'.

  • Open matlab as sudo.

  • In Matlab, navigate to usr/share/VPixx_Software_Tools/DatapixxToolbox_trunk/DatapixxToolbox/DatapixxDemos. It is useful to run DatapixxM16Demo first. On my system, this then tells me to run 'BitsPlusImagingPipelineTest(0);' once. This script basically tries to figure out the configuration of the DataPixx for your specific combination of graphics card, graphics driver and ViewPixx monitor.

  • Next, run 'BitsPlusIdentityClutTest;'. This did not work initially, so I called it again as 'BitsPlusIdentityClutTest ([],1);'. The second argument does some additional tests for the DataPixx.

  • With these last two steps in Matlab, make sure you read the output carefully!

PsychoPy

 

  • The instructions on the PsychoPy website were not very helpful (you should be able to get psychopy through neurodebian by simply calling 'sudo apt-get install psychopy'). However, there seemed to be lots of missing packages and I just couldn't get it to work. In the end, I found this blog post very useful: https://www.marsja.se/how-to-install-pyschopy-on-linux-mint-debian-edition/. I will reiterate the steps here, as I've had to change them slightly (they may be somewhat out of date).

  • The assumption is that you have already got access to the appropriate neurodebian repos, but if not, go to the neurodebian website, choose the software icon at the top. Now go to the 'psychophysics' software link. Scroll down to 'psychopy' and open. Here you can choose the server from which you want to install. Simply copy these commands and execute. Follow this with sudo apt-get update.

  • Next:

    • sudo apt-get install psychopy python-wxgtk3.0 python-yaml python-psutil

    • sudo apt-get install python-tables python-yaml python-pyo python-pygame python-pandas (the last one was critical in the end!)

  • Download AVbin: http://avbin.github.io/AVbin/Download.html and navigate to the folder where you have it stored. Call: sudo sh ./install-avbin-linux-x86-64-v10 (you may need to adjust the version number in the future).

  • Now PsychoPy is running at least, but I still couldn't get the most basic demo ('Hello World') to work. It was missing a module 'json_tricks'. I downloaded this from: https://pypi.python.org/pypi/json_tricks. Unzip the file and extract to a preferred directory (I've created a folder for 'External Modules' in /home/el/Documents/PsychoPy/). Next, open up PsychoPy and go to File-Preferences. Add the path '/home/el/Documents/PsychoPy/ExternModules/json_tricks-3.11.2/' to the 'paths' field in the 'General' tab.

  • Note that the demo only runs properly when starting PsychoPy as sudo.

  • Further packages were needed for a more complex/complete experiment that included eye tracking.

    • ioHub dependencies: sudo python -m pip install: psutil, msgpack, greenlet, gevent, numexpr, pytables, pyYAML, python-xlib.

    • more useful packages for ioHub: sudo python -m pip install pyserial, pyparallel, egi, iolabs, pyxid, xlrd.

    • PyGaze dependencies: sudo apt-get install python-numpy, python-scipy, python-matplotlib, python-imaging, python-pygame, psychopy, python-pyglet.

    • Disable the 'pyo' audio device in the PsychoPy preferences, and specify ['pygame','pysoundcard'].
